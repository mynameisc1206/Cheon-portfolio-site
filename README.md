# 📑 천웅재(Cheon Woong-jae) 개인 포트폴리오 사이트
본 프로젝트는 2025년 9월부터 2026년 3월까지 진행된 정부지원 교육과정의 결과물들을 취합하고, 개발자로서의 정체성을 시각화하기 위해 제작되었습니다.

**이력서履歷書란 무엇인가?**

: *어떤 이가 살아오면서 이룩한 학업이나 종사했던 직업 따위의 발자취를 적은 문서.*

포트폴리오 사이트 또한 새로운 형태의 이력서라 나는 생각한다. 이 사이트 또한 나를 보여주는 하나의 수단이기 때문이다. 이력서를 화려하게 꾸민다고 내가 어떤 사람인지, 내가 어떤 마음가짐을 가지고 회사에 지원을 하였는지를 알릴 수 있을까?, 나의 과거, 기술,  교육 경험들을 다른 사람에게 알릴 수 있는가, 사이트를 만드는 본인은 "아니다" 라 대답하고 싶다, 그렇기에 화려함을 빼고 욕심을 비우고 하나의 문서를 만든다는 마음가짐으로 프로젝트에 임하였다.

## 🔧 기능별 핵심 구현 설명

### 1. 지능형 헤더 및 스크롤 내비게이션 (`headerScroll.js`)
사용자의 스크롤 위치를 추적하여 최적의 내비게이션 경험을 제공합니다.

* **구현 목표**: 현재 탐색 중인 섹션을 시각적으로 피드백하고, 부드러운 페이지 이동 구현
* **핵심 로직**:
    * *Intersection Observer API*: 각 섹션(`.point`)의 가시성을 감시하여, 사용자가 보고 있는 영역에 맞춰 내비게이션 메뉴(`navPoint`)에 `active` 클래스를 자동으로 동기화합니다.
    * *Smooth Scrolling*: `scrollIntoView` 메서드를 활용하여 클릭 시 해당 섹션으로 부드럽게 이동하는 UX를 구현했습니다.
    * *반응형 상태 제어*: 윈도우 너비에 따라 헤더가 변화하는 임계값(Breakpoint)을 동적으로 설정하여 모바일과 데스크톱 환경 모두에서 안정적인 디자인을 유지합니다.

### 2. 프로젝트 상세 팝업 시스템 (`popup.js`)
별도의 페이지 이동 없이 프로젝트의 상세 정보를 집중도 있게 전달합니다.

* **구현 목표**: 모달 UI를 통한 콘텐츠 가독성 향상 및 사용자 흐름 유지
* **핵심 로직**:
    * *데이터 바인딩*: `data-popup` 속성을 이용해 클릭한 카드와 특정 팝업 콘텐츠를 1:1로 매칭하여 동적으로 노출합니다.
    * *스크롤 자동 포커싱*: 팝업이 활성화될 때 `scrollIntoView`를 통해 팝업의 최상단(Head)으로 화면을 자동 조정하여 정보 탐색의 편의성을 높였습니다.
    * *이벤트 전파 방지*: `stopPropagation()`을 적용하여 닫기 버튼 클릭 시 부모 요소로 이벤트가 전달되어 발생하는 의도치 않은 버그를 차단했습니다.

### 3. 실시간 숫자 카운팅 기술 스택 차트 (`skil-sect-chart.js`)
정적인 수치를 역동적인 애니메이션으로 표현하여 시각적 몰입감을 높였습니다.

* **구현 목표**: 기술 숙련도를 효과적으로 강조하기 위한 동적 시각화 구현
* **핵심 로직**:
    * *Mutation Observer API*: 탭 전환 시 요소의 `class` 변화를 감지하여, 해당 섹션이 노출되는 시점에만 애니메이션이 실행되도록 효율적인 상태 감시 로직을 구축했습니다.
    * *Random Text Shuffle*: 목표 수치에 도달하기 전까지 숫자를 무작위로 셔플하는 효과를 주어 텍스트가 완성되어가는 과정을 역동적으로 묘사했습니다.
    * *Performance Optimization*: `requestAnimationFrame`을 사용하여 브라우저 주사율에 최적화된 매끄러운 카운팅 애니메이션을 구현했습니다.

### 4. 선언적 탭 메뉴 시스템 (`tabMenu.js`)
제한된 공간 내에서 많은 정보를 효율적으로 분류하여 제공합니다.

* **구현 목표**: 직관적인 카테고리 전환을 통한 정보 응집도 향상
* **핵심 로직**:
    * *상태 초기화 및 활성화*: `forEach`문을 활용해 기존 활성화 상태를 일괄 제거한 뒤, 클릭된 요소의 `data-tab`과 일치하는 본문만 노출시키는 명확한 조건부 렌더링을 채택했습니다.
    * *안전한 변수 관리*: `let` 키워드를 사용하여 변수의 오염을 방지하고 코드의 가독성과 유지보수성을 확보했습니다.
